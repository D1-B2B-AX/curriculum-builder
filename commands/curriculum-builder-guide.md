# curriculum-builder 사용 가이드

## 이 도구는 왜 만들어졌나요?

기업 AI 교육 제안서를 작성할 때 LD가 반복적으로 수행하는 **"이 직무에 이 툴로 어떤 커리큘럼을 짤까"** 고민을 자동화하기 위해 만들어졌습니다.

회사·직무·툴·주제만 입력하면 직무 업무 조사부터 실습 task 선정, 모듈 4개(M1~M4) 구성까지 자동으로 진행되어 제안서 초안 작업 시간을 줄여줍니다.
메인 실습(M4)부터 역산으로 설계되기 때문에 "왜 배우는지 → 기초 → 실전"이 자연스럽게 이어지는 구조가 만들어집니다.

---

## 처음에 어떻게 시작하나요?

대화창에 아래 명령만 입력하면 됩니다:

```
/curriculum-builder
```

별도 파일·인자는 필요 없습니다. 대화로 필요한 정보를 질문드립니다.

### 미리 준비해두면 좋은 정보

**필수 (없으면 진행 불가)**:
- 회사명 (예: LG전자)
- 직무 (예: 온라인기획팀 담당자)
- 툴 (예: ChatGPT, 또는 ChatGPT + Claude Code처럼 복수 가능)
- 주제 (예: 데이터 분석 및 대시보드 구축)

**선택 (없으면 기본값 적용)**:
- 시수 / 보안 제약 / 수강생 수준
- ※ 현재는 이 정보를 수집만 하고 커리큘럼 생성에는 반영하지 않습니다. 아래 "현재 한계" 참고.

---

## 각 단계가 끝나면 어떻게 하나요?

전체 흐름 중 **4번의 확인 포인트**가 있습니다. 각 지점에서 결과를 보고 다음 중 하나를 선택하시면 됩니다:

- **"진행" / "OK"** → 다음 단계로 넘어감
- **"이 부분 수정해줘"** → 해당 단계만 다시 실행 (필요 시 후속 단계 연쇄 재실행)
- **"중단"** → 여기서 멈춤

### 확인 포인트 4곳

1. **입력 확정 후** — 회사·직무·툴·주제가 맞게 정리되었는지
2. **직무 task 목록 확인 후** — 파악된 직무 업무가 실제와 맞는지
3. **top3 실습 task 선정 후** — 메인 실습 후보 3개 + 주목할 task
4. **최종 커리큘럼 검토 후** — `curriculum.md` 검토

---

## 최종 결과물은 어떻게 보나요?

### 파일 위치

실행할 때마다 새 폴더가 생성됩니다:

```
curriculum-builder-output/
└── {회사명}_{직무}_{YYYYMMDDHHMM}/
    └── curriculum.md   ← 최종 결과물 (여러분이 보실 파일)
```

같은 폴더 안에 중간 산출물(`input.json`, `tasks.json`, `top3-tasks.json`, 모듈별 md 등)도 저장되지만, 검토·제안서 활용 용도로는 `curriculum.md` 하나면 충분합니다.

### md 파일 잘 보는 방법

텍스트 에디터로 그냥 열면 마크다운 문법이 그대로 보여 가독성이 떨어집니다. 아래 중 하나를 추천합니다:

- **VS Code 미리보기**: 파일 열고 `Ctrl+Shift+V` (또는 우측 상단의 미리보기 아이콘)
- **Typora·Obsidian 같은 마크다운 에디터**: 파일 열면 바로 렌더링됨
- **Claude Code 대화창**: 최종 조합 완료 직후 결과 요약이 대화창에 렌더링되어 표시되므로 별도 파일을 열지 않고도 확인 가능

### 더 나은 가시성을 위한 후속 계획

현재는 md 단일 출력이지만, **추후 HTML 출력 추가**를 검토하고 있습니다. HTML로 생성되면 브라우저에서 바로 렌더링·인쇄·PDF 변환 등이 쉬워집니다. Phase 2(보안·시수·수준 반영) 단계와 함께 추가될 예정입니다.

---

## 중간에 수정하고 싶으면요?

**언제든 수정 요청 가능합니다.** 진행 중·완료 후 상관없이 대화창에 말씀하시면 됩니다:

- "top1 task를 notable의 X로 바꿔줘" → top3 재선정
- "M4 실습 시나리오 다시 짜줘" → M4 재설계 (M3·M2·최종 조합 연쇄 재생성)
- "M3의 기초 실습 간결하게" → M3만 재생성
- "다시 처음부터" → `/curriculum-builder` 재호출 (새 폴더 생성)

---

## 현재 한계 (Phase 1)

- **보안·시수·수강생 수준 제약은 이 단계에서 반영하지 않습니다.** 이 3가지 제약을 실제로 커리큘럼에 녹여내는 Phase 2(Skill 9) 재단 로직은 본격 설계 전 단계입니다. 필요하시면 Phase 1 결과를 받으신 뒤 Claude에게 "이 curriculum.md를 N시간·파일 업로드 차단 환경에 맞춰 재단해줘"처럼 직접 요청하시면 수동 재단 가능합니다.

- **터미널에서 글자가 깨져 보이면** 이렇게 Claude Code에 다시 요청해보세요: **"글자가 깨져 보이니까 이모지·특수문자 대신 [완료], [경고], `->` 같은 ASCII로 바꿔서 다시 출력해줘"** — 같은 내용을 깨끗하게 다시 볼 수 있습니다.
