# curriculum-builder 사용 가이드

## 이 도구는 무엇인가요?

기업 AI 교육 제안서를 작성할 때 LD가 반복적으로 수행하는 **"이 회사의 이 직무에 이 툴로 어떤 커리큘럼을 짤까"** 고민을 자동화하는 오케스트레이터 스킬입니다.

회사·직무·툴·주제만 입력하면 — 직무 업무 조사 → 실습 task 선정 → 모듈 4개(M1 툴 소개 / M2 When·Why / M3 기초 실습 / M4 메인 실습) 구성 → 툴 feature 현재 시점 검증 → 표준 시수 두 세트(short 6시간 / long 12시간) 산출까지 자동으로 진행되어, **LD가 참고할 수 있는 표준 커리큘럼 초안**이 나옵니다.

메인 실습(M4)부터 역산으로 설계되기 때문에 "왜 배우는지(M2) → 기초(M3) → 실전(M4)"이 자연스럽게 이어집니다.

### 두 단계로 진행됩니다

| 단계 | 무엇 | 산출 |
|---|---|---|
| **Phase 1 (기본 설계)** | 시수·보안·수준 제약 없이 "이 직무를 이 툴·주제로 교육한다면 이상적으로 무엇을 배워야 하나"를 설계 | `phase1/curriculum.md` (중간본) |
| **Phase 2 (표준화)** | feature 현재 시점 검증 + 표준 시수 두 세트(short 6h·long 12h) + 톤앤매너 정리 + 모듈별 LD 설명 | `curriculum-final_{회사}_{직무}_{타임스탬프}.md` ← **LD가 받는 최종본** |
| Phase 3 (운영 제약 반영) | 기업 맥락·시수·보안 제약·수강생 수준을 반영해 고도화 | LD가 Claude와 자유 대화로 진행 (도구 범위 밖 — 아래 참고) |

`/curriculum-builder`는 **Phase 2까지 자동 완성**합니다. 시수·보안·수준 같은 운영 제약은 Phase 2의 표준 커리큘럼에는 반영되지 않고(표준 grid 고정: short = M1·M2 1h / M3·M4 2h, long = 2h/4h), Phase 3 자유 대화에서 LD가 직접 반영합니다.

## 설치하고 업데이트하기

### 처음 설치

Claude Code 대화창에 이렇게 입력하세요:

```
gh repo clone D1-B2B-AX/curriculum-builder 를 받고, 그 안의 install.ps1 을 실행해줘
```

그러면 Claude가 레포를 받고 `install.ps1` 을 돌려서 스킬을 `~/.claude/` 에 설치합니다. 설치가 끝나면 새 Claude Code 세션에서 `/curriculum-builder` 를 입력하면 됩니다.

### 업데이트

`/curriculum-builder` 를 실행하면 **매번 자동으로 최신 배포본인지 확인**합니다. 업데이트가 있으면 받는 방법을 그 자리에서 안내해드리니, 안내대로 따라하시면 됩니다. (업데이트 없이 현재 버전으로 그냥 쓰려면 "진행"이라고 답하시면 됩니다.)

## 처음에 어떻게 시작하나요?

설치가 끝났으면, 대화창에 아래 명령만 입력하면 됩니다:

```
/curriculum-builder
```

별도 파일·인자는 필요 없습니다. 대화로 필요한 정보를 질문드립니다.

### 미리 준비해두면 좋은 정보

**필수 (없으면 진행 불가)**:
- 회사명 (예: LG생활건강)
- 직무 (예: 디자이너, 온라인기획팀 담당자)
- 툴 (예: ChatGPT — 또는 Midjourney + Nano Banana + Figma 처럼 복수 가능)
- 주제 (예: AI 활용 디자인 실무 역량 향상)

**선택 (없으면 기본값 적용)**:
- 시수 / 보안 제약 / 수강생 수준
- ※ 이 3가지는 입력해두면 좋지만, **Phase 2까지의 표준 커리큘럼에는 반영되지 않습니다.** Phase 3 자유 대화 단계에서 활용됩니다 (아래 "현재 단계와 Phase 3" 참고).

## 진행 중 어떤 지점에서 확인하나요?

전체 흐름 중 **시스템이 멈춰서 LD 응답을 받는 확인 포인트는 4곳**입니다. 그 사이 단계들(직무 task 도출 · 툴 기능 조사 · M1~M4 설계 · 공정 1~3)은 자동으로 흘러가고 "확인해주세요"로 멈추지 않습니다 — 다만 **언제든 끼어들어 수정을 요청할 수 있습니다.**

### 확인 포인트 4곳

1. **입력 확정 후** — 회사·직무·툴·주제가 맞게 정리되었는지
2. **실습 task 후보 선정 후** — 툴별로 후보 **5개**(메인 실습으로 가치 있는 것 위주, 일부는 "잠재 후보")를 제시합니다. 그중 **메인 실습(M4)으로 설계할 task를 직접 선택**하시면 됩니다.
   - "진행" / "기본" 등으로만 답하면 → 1위 task로 자동 선정
   - 다른 후보로 바꾸고 싶으면 → 후보 번호를 말씀하시면 됩니다
3. **Phase 1 조합 후** — `phase1/curriculum.md` (Phase 1 중간본) 검토. 여기가 **M1~M4 모듈 내용을 수정하는 정식 자리**입니다 — "M3 기초 실습 더 간결하게", "M4 시나리오 다르게" 같은 요청을 하면 해당 모듈만 다시 설계되고 이후 단계가 연쇄 재실행됩니다.
4. **최종 커리큘럼 완성 후 (Phase 2 끝)** — `curriculum-final_{회사}_{직무}_{타임스탬프}.md` 검토. 이때 **"기업 맥락·시수·보안 제약·수강생 수준을 반영해서 고도화하는 대화를 계속하시겠습니까?"** 라고 묻습니다 — "예"면 Phase 3 자유 대화로 진입하고, "아니오"면 여기서 종료됩니다.

각 확인 포인트에서 — "진행"이면 다음으로, "이 부분 수정해줘"면 해당 단계만 다시 (필요 시 후속 연쇄), 질문이 있으면 답을 먼저 드린 뒤 다시 여쭙니다.

## 최종 결과물은 어떻게 보나요?

### 파일 위치

실행할 때마다 **바탕화면에 새 폴더**가 생성됩니다:

```
바탕화면\curriculum-builder-output\
└── {회사명}_{직무}_{YYYYMMDDHHMM}\
    ├── curriculum-final_{회사}_{직무}_{타임스탬프}.md   ← ★ LD가 받는 최종본 (폴더 열면 바로 보임)
    ├── input.json · tasks.json · tool-features.json · top-tasks.json   (메타 자료)
    ├── phase1\
    │   ├── m1~m4-{툴}.md · curriculum.md   (Phase 1 모듈 기획 + 중간 조합본)
    │   └── task-research\   (직무 업무 조사 5개 파일)
    └── phase2\
        ├── factcheck\ · blocks\ · block-refinement\   (Phase 2 공정 1~3 작업물)
        └── curriculum-final-meta\   (final-result.json · self-check-result.json — 검증 자료)
```

- **`curriculum-final_{회사}_{직무}_{타임스탬프}.md`** = LD가 검토·제안서에 쓰는 메인 문서. 안에 툴별로 ① 커리큘럼 표 — long(12h) ② 커리큘럼 표 — short(6h) ③ 모듈별 LD 설명(M1~M4 각 5~7줄)이 들어 있습니다.
- `phase1\curriculum.md` = Phase 1 중간 조합본 (표준 시수 및 검증 전 단계). 참고용으로만 보시면 됩니다.
- 나머지 폴더의 파일들은 작업 과정 산출물이라, 직접 보실 때에는 `curriculum-final_{회사}_{직무}_{타임스탬프}.md` 파일 위주로 보시면 됩니다.

### md 파일 잘 보는 방법

텍스트 에디터로 그냥 열면 마크다운 문법이 그대로 보여 가독성이 떨어집니다. 아래 중 편한 방법을 쓰세요:

- **VS Code 내장 미리보기** — 파일 열고 `Ctrl+Shift+V` (또는 우측 상단 미리보기 아이콘). 별도 설치 없이 바로 됩니다.
- **VS Code 확장 — Markdown All in One** — 목차 자동 생성, 단축키, 테이블 정렬, 체크박스 토글 등 마크다운 편집·읽기 보조 기능이 풍부합니다. (마켓플레이스에서 "Markdown All in One" 검색 → 설치)
- **VS Code 확장 — Markdown Preview Enhanced** — 내장 미리보기보다 강력한 렌더링 + PDF·HTML로 내보내기 기능. 제안서에 표를 깔끔하게 옮길 때 유용합니다.
- **Obsidian** — 무료 마크다운 에디터. 파일 열면 곧바로 렌더링되어 마크다운 문법을 아예 안 봐도 됩니다.

## 중간에 수정하고 싶으면요?

**언제든 수정 요청 가능합니다.** 진행 중이든 완료 후든, 대화창에 말씀하시면 됩니다. 어떤 부분을 바꾸든 그 영향을 받는 다음 단계들이 자동으로 다시 만들어지니, "어디부터 다시 해야 하지" 신경 쓰실 필요는 없습니다.

자주 쓰는 요청 예시:

- "이 task 말고 다른 task로 바꿔줘" — 메인 실습 task를 바꾸고, 그에 맞춰 모듈들이 다시 짜집니다
- "M4 실습 시나리오를 다르게 짜줘" — 선정된 task는 그대로 두고 실습 흐름만 다시 짜집니다 (task 자체를 바꾸고 싶으면 위의 "다른 task로 바꿔줘"라고 하세요)
- "M3 기초 실습을 더 간결하게" — M3 모듈이 다듬어집니다
- "M2 꼭지를 다르게" — M2 꼭지가 다시 짜집니다
- "다시 처음부터" — `/curriculum-builder` 를 다시 입력하세요 (새 폴더가 생깁니다)

## 현재 단계와 Phase 3

- **Phase 1·2는 이 도구가 자동 완성합니다.** 산출물은 "LD가 받는 표준 원형 커리큘럼"입니다.
- **Phase 3 (기업 맥락·시수·보안·수강생 수준 반영)는 LD-Claude 자유 대화 영역**입니다 — 도구가 자동으로 하지 않습니다. 최종 커리큘럼 완성 후 "고도화 대화를 계속하시겠습니까?"에 "예"라고 답하면 진입하고, 이후 자유롭게 요청하시면 됩니다.
