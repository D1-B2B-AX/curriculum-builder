# Golden Rule and Splitting

## Golden Rule

> If the **immediate deliverable** is different, **split**.
> If the **immediate deliverable** is the same, **keep as one task**.

The "immediate deliverable" is the tangible output produced at the end of the action.

## When to split

Split when:
- Two verbs produce **different deliverables**: "보고서를 작성하고 경영진에게 보고한다" → (1) 보고서 작성 (2) 경영진 보고 — deliverables: 문서 vs 전달 행위
- Actions belong to **different phases**: "시장을 조사하고 전략을 수립한다" → (1) 시장 조사 (SENSE/P1) (2) 전략 수립 (DECIDE/P2)
- A clear **handoff** exists between steps: "데이터를 수집하여 대시보드에 반영한다" → (1) 수집 (2) 반영

## When NOT to split

Keep as one task when:
- **Object enumeration** with same action: "예산, 인력, 자원을 관리한다" → 1 task. All objects share "관리" as the action and produce the same type of deliverable.
- **Sequential sub-steps** producing a single output: "데이터를 수집·정리·분석한다" → 1 task if the output is a single analytical result
- **Verb repetition** with same scope: "모니터링하고 관리한다" → 1 task if they describe the same ongoing oversight

## Object enumeration rule

When a task lists multiple objects (A, B, C) under one verb:
- If all objects are managed the same way with the same deliverable → **keep as one**
- If objects require fundamentally different processes → **split by process**

Example:
- Keep: "카드 상품별(M/X/Zero) 기획·운영을 관리한다" → same management process
- Split: "카드 상품을 기획하고 고객 민원을 처리한다" → different processes entirely

## Over-splitting avoidance

Signs of over-splitting:
- Resulting atomic tasks are too small to be independently meaningful
- Resulting tasks are sub-steps of a single coherent action
- Split produces 4+ tasks from one raw task (review carefully)

Rule of thumb: 1 raw task → 1-3 atomic tasks. If producing 4+, reconsider.

## Immediate deliverable test

For each candidate split, ask:
1. Does this sub-task produce a **distinct, nameable output**?
2. Could this sub-task be **assigned to a different person**?
3. Does this sub-task have a **different phase/primitive**?

If yes to 2+ questions → split is justified.
