# curriculum-builder 사용 가이드

`/curriculum-builder-guide`를 호출했을 때 Claude가 LD에게 안내할 가이드 내용입니다.

---

당신은 curriculum-builder 스킬 패키지의 **사용 가이드**를 LD에게 친절하게 안내합니다.
아래 내용을 **그대로 출력**하세요 (임의 요약·축약 금지).

```
# curriculum-builder 사용 가이드

**무엇을 하는 도구?**
회사·직무·툴·주제를 입력하면 M1(툴 소개) → M2(활용 맥락) → M3(기초 실습) → M4(메인 실습) 4개 모듈로 구성된 교육 커리큘럼을 자동 생성합니다.
메인 실습(M4)부터 역산으로 설계되어 "왜 배우는지 → 기초 → 실전"이 자연스럽게 이어지는 구조입니다.

---

## 1. 실행 방법

대화창에 다음을 입력하세요:

```
/curriculum-builder
```

별도 파일·인자 없이 호출만 하면 됩니다. Claude가 대화로 필요한 정보를 수집합니다.

---

## 2. 미리 준비할 정보

### 필수 (없으면 진행 안 됨)
- **회사명**: 예) "LG전자"
- **직무**: 예) "온라인기획팀 담당자"
- **툴**: 예) "ChatGPT", "ChatGPT + Claude Code" (복수 가능)
- **주제**: 예) "데이터 분석 및 대시보드 구축"

### 선택 (없으면 기본값 적용)
- **시수**: 미입력 시 8시간
- **보안 제약**: 예) "외부 AI 사이트 접근 불가, 파일 업로드 안 됨"
- **수강생 수준**: 예) "ChatGPT 기본 사용 경험 있음"

※ 선택 정보는 Phase 2(시수·보안·수준 재단) 단계에서 사용됩니다. Phase 1 커리큘럼 생성 자체에는 영향이 없습니다.

---

## 3. 실행 흐름

1. **입력 수집** — Claude가 LD에게 회사·직무·툴·주제 등 질문
2. **직무 task 도출** — 직무가 실제로 하는 업무 15~25개 파악
3. **툴 기능 조사** — 입력한 툴의 주요 기능 웹 검색 (복수 툴이면 각각)
4. **실습 task top3 선정** — 평가 축(툴 적합성·AI 변화 폭·직무 중요도)으로 점수화
5. **M4 메인 실습 설계** → **M3 기초** → **M2 When/Why** → **M1 툴 소개** (역산)
6. **최종 커리큘럼 조합** — `curriculum.md` 단일 파일 생성

중간에 4번의 확인 포인트가 있습니다:
- 입력 확정 후 / task 목록 확인 후 / top3 선정 후 / 최종 커리큘럼 검토

각 포인트에서 "OK" / "수정 요청" / "중단" 중 선택 가능.

---

## 4. 산출물 위치

실행할 때마다 새 폴더가 생성됩니다:

```
curriculum-builder-output/
└── {회사명}_{직무}_{YYYYMMDDHHMM}/
    ├── input.json                   # 수집된 입력
    ├── tasks.json                   # 직무 task 목록
    ├── tool-features.json           # 툴 기능 조사 결과
    ├── top3-tasks.json              # 선정된 top3 task + 근거
    ├── m1-{tool}.md ~ m4-{tool}.md  # 모듈별 구성 기획 (툴별)
    ├── curriculum.md                # 최종 커리큘럼 (메인 산출물)
    └── task-research/               # task 도출 세부 산출물
```

**가장 중요한 파일: `curriculum.md`** — LD가 검토하고 제안서에 옮길 최종 결과물입니다.

---

## 5. 수정 요청 방법

언제든 수정 가능:

- **"top1 task를 notable의 X로 바꿔줘"** — top3 재선정
- **"M4 실습 시나리오 다시 짜줘"** — M4 재설계 (M3·M2·최종 조합 연쇄 재생성)
- **"M3의 기초 실습 간결하게"** — M3만 재생성
- **"다시 처음부터"** — `/curriculum-builder` 재호출 (새 폴더 생성)

---

## 6. 현재 한계 (Phase 1)

- **보안·시수·수강생 수준 제약은 이 단계에서 반영하지 않습니다** (Phase 2 Skill 9에서 처리 예정, 본격 설계 전)
- **HTML 출력 없음** — 현재는 md 단일 출력. HTML은 Phase 2 이후 추가 예정
- **서브 스킬 LD 출력에 일부 이모지·특수문자** 포함 — 터미널 렌더링 깨질 수 있음. ASCII 대체는 후속 과제

---

## 7. 문제 생겼을 때

- **스킬이 보이지 않음** → `install.ps1` 재실행 or 수동으로 `skills/`·`commands/` 폴더 복사 확인
- **Skill 2 에서 멈춤** → 건호님 5개 스킬(company-role-task-research 외 4개)이 모두 설치되어 있는지 확인
- **한글 깨짐** → 터미널 인코딩이 UTF-8인지 확인 (`chcp 65001`)
```
