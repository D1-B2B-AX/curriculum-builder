# Phase 1 Module Heading Schema

Phase 1 (Skill 5~8)이 생성하는 m1/m2/m3/m4 모듈의 **실제 헤딩 스키마** (Skill 원본 파일에서 추출).
Plan v2 공정 1 (Tool Feature Factcheck)의 Step S6가 재구성 시 본 스키마를 유지해야 한다.
Validation v2 기준 3 (C) Pre-check의 기준 문서.

> 추출 원천: `~/.claude/skills/curriculum-builder/skills/skill-5-m4-design.md`, `skill-6-m3-design.md`, `skill-7-m2-design.md`, `skill-8-m1-design.md`의 "## Output: m{N}-{tool}.md > ### 파일 구조" 섹션.
> ★ v4 정정 (2026-05-04): "## Skill 9 인계" 섹션 영역은 **Phase 1 영역에서 삭제** 본질 (Phase 3 영역 — 본 단계 산출물에 박지 X). 섹션명 영역은 Skill 원본 영역 정합 유지.

## 공통 규약

- 각 파일 최상단에 생성 정보 blockquote(`> 생성 정보` 블록) 존재. **헤딩 diff 대상이 아님** (정규식 `^(#{1,6})\s+(.+)$` 매치에서 제외).
- 헤딩 레벨은 `#` (타이틀, 파일당 1개) / `##` (섹션) / `###` (서브섹션).
- 섹션명은 Skill 원본 그대로 유지 — `## 다음 단계 (Skill 간 인계)` 등 이름 변경 금지. **`## Skill 9 인계` 영역은 v4 정정 후 Phase 1 영역에서 삭제** (산출물 영역 검증 시 본 영역 박혀있으면 결함).

## 전역 규칙

- 헤딩 레벨(`# / ## / ###`)은 원본과 동일하게 유지.
- 생성 정보 blockquote(`> 생성 정보 ...`)는 헤딩 비교 대상이 **아님** (헤딩 정규식 `^(#{1,6})\s+(.+)$` 매치에서 제외).
- M4의 `## 실습 시나리오` 내부 numbered list는 **헤딩이 아니라 본문 내용**. 재구성 시 step 수 비교는 본문의 `^\s*\d+\.\s+\*\*` 정규식 카운트 기반으로 수행한다.
- M3의 `## 기초 실습 시나리오` 하위 `### 실습 N` 서브헤딩 개수는 M4의 "사용 feature 목록" 표 행 수와 **일치**해야 한다 (역산 원칙).
- M2의 `## 강의 구성 제안 (M2 모듈의 주 본문)` 하위 `### 꼭지 N` 서브헤딩 개수는 **3~5 범위**.
- M1의 `## 강의 구성 제안 (M1 모듈의 주 본문)` 하위 `### 꼭지 N` 서브헤딩 개수는 **정확히 3개**.
- Phase 2 공정 1의 Step S6 재구성 프롬프트는 본 스키마를 **참조**하되, 원본 `m{N}-{tool}.md`의 실제 헤딩 구조를 **1차 기준**으로 삼아야 한다 (원본이 본 스키마보다 추가 섹션이 있으면 원본 유지, 누락되어 있으면 원본 유지).

## Level별 재구성 규칙 (prompts_reconstruction.md와 동기화)

### Level 1 (M1/M2 경량 교체)
- 본문 문자열만 수정. 모든 헤딩·순서·서브헤딩 변경 금지.
- feature 언급 블록만 new_feature로 치환.

### Level 2 (M3/M4 feature만 교체)
- 헤딩·순서 불변.
- M4는 `## 실습 시나리오` 내부 numbered list의 step 수(정규식 `^\s*\d+\.\s+\*\*`)가 원본과 동일.
- M3는 `### 실습 N` 서브헤딩 수가 원본과 동일.

### Level 3 (M3/M4 실습 세부 재작성)
- 헤딩·순서 불변.
- `## 학습 목표` 섹션 본문 및 M4의 `## 실습 task 선정 근거` 상단 "**선정**: Top {rank} — `{task_one_liner}`" 문구는 원본과 **정확히 일치** (task 불변).
- M4 타이틀의 `{task_one_liner}` 부분도 불변.
- step 재작성은 `## 실습 시나리오` 내부 numbered list 본문에서 수행. step 수 변경은 `level_selection_basis`에 근거 필수.

### behavior_update_reflection (STEP 8 경량 반영)
- 본문 내 문장·주석 추가만 허용.
- 헤딩·순서·feature name 모두 변경 금지.
- change_delta의 핵심 명사구(3단어 이상 연속 일치) 최소 1개를 본문에 삽입.
