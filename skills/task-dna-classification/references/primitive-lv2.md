# Primitive-LV2 상세 레퍼런스

> Step ③ IVC DNA 정밀 분석 시 Primitive-LV2 선택의 정의·경계·DoD를 제공한다.
> TRANSFORM은 M2(Content, 디지털) 세트만 수록. M1(Form, 물리) Primitive-LV2는 Part 2에서 이름만 제시(Fabricate~FinishProtect 6종). Mechanism-LV2는 `mechanism-lv2.md` 참조.

## A.1 SENSE (관찰·수집) — 6종

**Phase:** P1 | **Mechanism (Primary):** M2 – Content

**목적:** 해석·집계·판단 없이 사실을 있는 그대로 기록

**포함 범위:** 원시 데이터/사건/발화/아티팩트의 원문·원형 수집/기록

**제외:**
- 의미 해석·요약·집계 → DECIDE / TRANSFORM.Structure
- 형식 변환·편집·정제 → TRANSFORM
- 전달/배포/정산 → TRANSFER
- 권한 부여·승인·서명·정책 공표 → COMMIT
- 기준 적합성·효과 검증 → ASSURE

**공통 DoD 체크리스트:**
타임스탬프(UTC+TZ), 출처/라인리지, 주체ID(PII는 토큰화/마스킹), 원문 payload, 스키마 적합, 중복 방지 키(idempotency)

**공통 KPI:** Coverage, Missing/dup %, Ingest latency, Field accuracy(단위/통화/타임존), Lineage 완결성, Consent/PII 태깅율

### 1) CaptureTelemetry (텔레메트리 수집)

| 항목 | 내용 |
|------|------|
| **정의** | 시스템/디바이스가 자동 발생하는 이벤트·로그·메트릭을 원형 그대로 적재 |
| **포함** | 앱/웹 이벤트, 서버·네트워크·DB 메트릭, 에러 로그, IoT 스트림 |
| **제외** | 지표 산출(DAU 등)·필터링·샘플링 정책 설계 → DECIDE/TRANSFORM |
| **DoD** | 원시 레코드(events_raw): ts, subject_id, event_name, payload, src, tz |
| **템플릿** | Trigger(신호 발생) → Input(payload/headers) → Action(append raw) → Output(events_raw) → Next(Structure/Monitoring) |
| **예시** | (교) 수강 영상 재생/완료 이벤트 수집 / (범) API 에러율 로그 |
| **KPI** | 커버리지, 적재지연, 시간동기 정확도, 중복률 |
| **Pitfalls** | 중복 삽입, 타임존 혼선, PII 노출 → idempotency key/NTP/마스킹 |

### 2) RecordTransaction (업무 사건 기록)

| 항목 | 내용 |
|------|------|
| **정의** | 비즈니스 상태 전이를 일으키는 사건을 고유 ID와 상태로 기록 |
| **포함** | CRM 리드 생성, 수강신청, 결제 Webhook, 티켓 생성 |
| **제외** | 인보이스 문서 생성 → TRANSFORM.Convert, 입금 처리 → TRANSFER.Settle |
| **DoD** | tx_id, ts, actor, type, amount/qty, currency, status |
| **템플릿** | Trigger(사건 발생) → Input(Webhook/폼) → Action(정규화·기록) → Output(tx_record) → Next(승인/정산) |
| **예시** | (교) "교육 문의"→ CRM 리드 레코드 / (범) POS 판매 기록 생성 |
| **KPI** | 누락/중복률, 필드 정확도, 기록 지연 |
| **Pitfalls** | 리트라이 중복, 통화/단위 혼재, 상태머신 미정의 |

### 3) MeasureField (현장 계측)

| 항목 | 내용 |
|------|------|
| **정의** | 사람이 SOP에 따라 물리 세계를 계측하고 원시값 기록 |
| **포함** | 온도·소음·밝기·치수, 설비 점검치, 환경 메타(위치/조건) |
| **제외** | 즉시 보정·수리 → TRANSFORM.Repair, 평가 리포트 → DECIDE/ASSURE |
| **DoD** | 측정 테이블: ts, device_id, calibration, unit, value, env_meta |
| **템플릿** | Trigger(점검계획/주기) → Input(SOP·장비) → Action(측정·기록) → Output(measurements) → Next(검증/알람) |
| **예시** | (교) 강의실 소음/프로젝터 밝기 측정 / (범) 공정 샘플 치수 측정 |
| **KPI** | 정밀도·재현성, SOP 준수율, 환경메타 기록률 |
| **Pitfalls** | 장비 미보정, 단위 혼재, 측정자 편향 |

### 4) ElicitAndRecordResponses (응답 유도·기록)

| 항목 | 내용 |
|------|------|
| **정의** | 설문/인터뷰/UT/콜 등 사람의 진술·행동을 유도·관찰·원문 기록 |
| **포함** | 설문 응답, 인터뷰 노트, 녹취 텍스트, 사용성 테스트 관찰 로그 |
| **제외** | 요약·코딩·테마 도출 → DECIDE.Interpret |
| **DoD** | 원응답/노트/녹취 텍스트 + 동의/PII 태그 |
| **템플릿** | Trigger(조사/콜 종료) → Input(질문지·동의) → Action(질문·관찰·기록) → Output(responses_raw) → Next(요약/해석) |
| **예시** | (교) 디스커버리 콜 원문 / (범) NPS 자유응답 원문 |
| **KPI** | 응답률, 표본 편향, 질문 완결성, 동의율 |
| **Pitfalls** | 유도 질문, 동의 누락, 현장 해석 섞기(DECIDE 침범) |

### 5) DiscoverAndCaptureExternal (외부 정보 탐색·취득)

| 항목 | 내용 |
|------|------|
| **정의** | 공개/허가된 외부 출처(웹/보고서/API)에서 사실·문서를 스냅샷/링크로 취득 |
| **포함** | 웹 크롤·스크랩, 뉴스·공시, 입찰 가이드 원문, 외부 API 원문 페이로드 |
| **제외** | 라이선스/접근권 부여 행위 → COMMIT.Authorization, 요약/정리 → DECIDE/TRANSFORM.Structure |
| **DoD** | 스냅샷·원문 캡처/링크 컬렉션(captured_at, source_url, license, hash) |
| **템플릿** | Trigger(리서치 필요/크론) → Input(URL/쿼리) → Action(취득) → Output(external_snapshot) → Next(요약/구조화) |
| **예시** | (교) 고객사 웹/뉴스 원문 캡처 / (범) 경쟁사 가격 페이지 스냅샷 |
| **KPI** | 최신성, 출처 신뢰도, 중복 제거율, 라이선스 준수 |
| **Pitfalls** | 크롤링 정책 위반, 출처 불명, 업데이트 누락 |

### 6) IngestInternalArtifacts (내부 아티팩트 인입) ＊추가

| 항목 | 내용 |
|------|------|
| **정의** | 조직 내부의 기존 산출물(위키·메일·드라이브·스캔본·슬랙 내보내기)을 등록·카탈로그화 없이 원문 그대로 수집 |
| **포함** | 내부 문서 파일, 메일 EML/MSG, 회의록 PDF, 이미지 스캔 원본 |
| **제외** | 메타데이터 보강/태깅/분류 → TRANSFORM.Structure, 접근권한 변경 → COMMIT.Authorization |
| **DoD** | 원문 파일+기본 메타(path/id, captured_at, owner, checksum, minimal_acl_ref) |
| **템플릿** | Trigger(케이스 착수/증거 수집) → Input(링크/첨부) → Action(원문 인입) → Output(artifacts_raw) → Next(카탈로그/태깅) |
| **예시** | (교) 과거 제안서 원본 일괄 인입 / (범) 계약 스캔본 원문 적재 |
| **KPI** | 인입 커버리지, 무결성(해시 일치), 권한 위반 제로, 중복율 |
| **Pitfalls** | 버전 혼선, 권한 상속 오류, 무단 PII 확산 |

### SENSE 경계 결정 트리

```text
비즈니스 상태 전이 사건?        → RecordTransaction
사람의 발화/행태?               → ElicitAndRecordResponses
사람이 장비로 계측?             → MeasureField
시스템이 스스로 방출?           → CaptureTelemetry
외부 출처 취득?                 → DiscoverAndCaptureExternal
내부 산출물 인입?               → IngestInternalArtifacts
```

## A.2 DECIDE (선택·계획) — 6종

**목적:** SENSE로 모은 사실을 구조화/해석해 '안(案)'을 만든다 (아직 권리·의무 없음)

**Mechanism Primary:** M2 – Content (Interpret/Structure)

### 1) Select (고르기)

| 항목 | 내용 |
|------|------|
| **정의** | 후보 대안 중 단 하나를 선택하고 사유를 남긴다 (M2-Interpret) |
| **포함** | 강사/벤더/도구 최종 선정 |
| **제외** | 계약 체결 → COMMIT-Contract |
| **DoD** | 결정 기록(선정 사유·근거·대안비교), 티켓/상태 변경 |
| **템플릿** | 트리거(후보 준비) → 입력(평가기준) → 행동(선정하기) → 산출물(결정 메모) → 다음(협상/계약) |
| **KPI** | 리드타임, 재선정률, 근거 충실도 |
| **Pitfalls** | 기준 불명, 이해상충 미기록, 데이터 미활용 |

### 2) Rank (우선순위 매기기)

| 항목 | 내용 |
|------|------|
| **정의** | 후보·백로그에 점수/순위를 부여한다 (M2-Interpret/Structure) |
| **포함** | 쇼트리스트 스코어링, 기능 백로그 정렬 |
| **제외** | 자원 배정 확정 → Allocate / 예산 확정 → COMMIT-Budget |
| **DoD** | 순위표(가중치·산식·원점수 포함), 재현 가능한 점수화 기준 |
| **KPI** | 재평가 변동(일관성), 이해관계자 만족도, 재작업률 |
| **Pitfalls** | 가중치 임의변경, 사후 정당화 |

### 3) Allocate (연속 자원 배분하기)

| 항목 | 내용 |
|------|------|
| **정의** | 예산/정원/시간 용량(capacity) 등 연속자원의 분배안을 수립한다 (M2-Interpret) |
| **포함** | 좌석/시간용량/예산 배분안, 인력 FTE 할당안 |
| **제외** | 예산·정원 시스템 확정 → COMMIT-Budget/Reservation |
| **DoD** | 제약조건·가정이 명시된 배분안(초안), 시나리오(옵션) 표 |
| **KPI** | 가동률, 과·부족률, 제약 위반/충돌 수 |
| **Pitfalls** | 제약 누락, 이중할당, 민감도 분석 부재 |

### 4) Schedule (시간·순서 계획하기)

| 항목 | 내용 |
|------|------|
| **정의** | 캘린더 이벤트 생성 전, 일정/순서를 설계한다 (M2-Structure) |
| **포함** | 마일스톤 맵, 주차 계획, 의존성/버퍼 설계 |
| **제외** | 캘린더 이벤트 생성·초대 발송 → TRANSFER-Distribute |
| **DoD** | 스케줄안(제약·의존성·버퍼), 충돌 검증 로그 |
| **KPI** | 충돌률, 변경횟수, 예측오차(납기 준수) |
| **Pitfalls** | 가용성 미확인, 버퍼 0, 크리티컬 패스 미도출 |

### 5) DefineTargets (목표·사양 정의하기)

| 항목 | 내용 |
|------|------|
| **정의** | SLO/SLA/OKR/가격대역 등 목표·사양을 측정가능하게 정의한다 (M2-Structure) |
| **포함** | 학습목표·평가방식·성능 임계값, 가격 밴드 |
| **제외** | 정책 공표/적용 → COMMIT-Policy |
| **DoD** | 타깃 스펙 문서(지표정의·측정방법·데이터소스·임계값) |
| **KPI** | 목표-실적 괴리, 변경 승인율, 측정가능성(명료도) |
| **Pitfalls** | 측정불가 지표, 모호한 정의, 소스 불일치 |

### 6) SpecifyRules (의사결정 규칙 명세하기)

| 항목 | 내용 |
|------|------|
| **정의** | 스코어카드·룰·임계값 등 의사결정 로직의 초안을 명세한다 (M2-Structure) |
| **포함** | 할인 룰, 평가 기준표, 라우팅 룰, 점수 컷오프 |
| **제외** | 룰의 정책화/공표/시행 → COMMIT-Policy |
| **DoD** | 룰 스펙(버전·테스트케이스·예외·데이터 편향 점검) |
| **KPI** | 커버리지, 오탐/미탐, 운영 복잡도 |
| **Pitfalls** | 예외 누락, 편향 반영, 테스트 미흡 |

### DECIDE 경계 규칙

- **Select vs Rank:** 산출물이 단일 결정이면 Select, 정렬 리스트면 Rank
- **Allocate vs Schedule:** 연속자원(돈/정원/시간용량) 분배 = Allocate, 캘린더 시간 슬롯/순서 배정 = Schedule
- **DefineTargets vs SpecifyRules:** 목표/성능 기준 = Targets, 의사결정 로직 = Rules
- **DECIDE vs TRANSFER/COMMIT:** 안(초안) 문서 = DECIDE, 발송 = TRANSFER, 확정/공표 = COMMIT

## A.3 TRANSFORM (제작·수정) — M2(Content) 세트 7종

> M1(Form, 물리) 세트는 Part 2 §2.1에 이름만 수록 (Fabricate, Assemble, Reshape, Repair-Physical, Disassemble, FinishProtect — 6종). 상세는 별도 제공 예정.

**목적:** 산출물을 만든다/바꾼다 (디지털·물리). 권리/소유/돈 이동은 없음.

**Primary Mechanism:** 디지털=M2(Content), 물리=M1(Form)

### 1) Generate (만들다, 0→1)

| 항목 | 내용 |
|------|------|
| **정의** | 없던 결과물을 새로 생성. 디지털=M2(Generate), 물리=M1(Create) |
| **포함** | 제안서·커리큘럼·데크·보고서 초안 만들기, 3D 프린팅로 부품 제작 |
| **제외** | N→1 결합은 Integrate, 포맷만 바꾸면 Convert |
| **DoD** | 새 아티팩트(소유자/버전/위치 명시) |
| **KPI** | 제작 리드타임, 초안→최종 전환율, 재사용률 |
| **Pitfalls** | 목적/수용기준 불명, 템플릿 미사용 |

### 2) Edit (다듬다, 의도 유지 개선)

| 항목 | 내용 |
|------|------|
| **정의** | 의도·내용 유지하며 품질 개선(가독성/성능/표현). M2(Refine) |
| **포함** | 윤문, 시각 보정, 코드 리팩토링 |
| **제외** | 결함 복구는 Repair, 포맷만 변경은 Convert |
| **DoD** | 개정본(v2) + 변경 로그 |
| **KPI** | 오류 감소, 가독성/성능 지표 개선 |
| **Pitfalls** | 원의도 훼손, 이력 미관리 |

### 3) Repair (고치다, 결함 복구/정상화)

| 항목 | 내용 |
|------|------|
| **정의** | 규격/요건 대비 결함을 식별·해결하여 정상 상태 복귀. 디지털=M2(Refine), 물리=M1(Modify) |
| **포함** | 오탈자·가격 오류 수정, 버그 핫픽스, 장비 수리/캘리브레이션 |
| **제외** | 검증만은 ASSURE.Verification |
| **DoD** | 수정본 + 원인/해결 로그 + 테스트 통과 |
| **KPI** | 재발률, MTTR, 고객 영향 감소 |
| **Pitfalls** | 근본원인 미해결(RCA 부재), 회귀 테스트 미흡 |

### 4) Convert (변환하다, 형식/매체 변경, 1→1)

| 항목 | 내용 |
|------|------|
| **정의** | 의미·내용 유지, 표현 매체/포맷만 변경. M2(Structure) |
| **포함** | DOC→PDF, CSV→XLSX, 오디오→텍스트(동일 내용의 전사), 모델 포맷 변환 |
| **제외** | 구조 재배열/스키마 변경은 Structure, 의미 수정은 Edit |
| **DoD** | 변환본 + 변환 로그(손실/호환성) |
| **KPI** | 변환 성공률, 호환성, 손실률 |
| **Pitfalls** | 폰트/도식 깨짐, 메타 손실 |

### 5) Integrate (결합하다, N→1)

| 항목 | 내용 |
|------|------|
| **정의** | 여러 소스/모듈을 하나로 합침(조립/병합/조인). 디지털=M2(Structure), 물리=M1(Combine) |
| **포함** | 다출처 데이터 조인, 문서 병합, 모듈 합본, 부품 조립 |
| **제외** | 1개 대상 내부 구조화는 Structure, 단순 전달은 TRANSFER |
| **DoD** | 통합본 + 계보(lineage)/키 충돌 해결 규칙 |
| **KPI** | 결합 정확도, 중복 제거율, 키 매칭율 |
| **Pitfalls** | 키 불일치, 최신본 기준 미정, 출처 추적 실패 |

### 6) Structure (구조화하다, 라벨·스키마·카탈로그화)

| 항목 | 내용 |
|------|------|
| **정의** | 메타데이터 부여/스키마 매핑/인덱싱 등 탐색·재사용을 위한 구조화. M2(Structure) |
| **포함** | 데이터 딕셔너리, 라벨링, 카탈로그, 정규화/표준 필드 매핑 |
| **제외** | N→1 결합은 Integrate, 의미 해석·판단은 DECIDE |
| **DoD** | 구조화 테이블/카탈로그 + 스키마 문서 + 접근권한 태깅 |
| **KPI** | 검색 적중률, 탐색성, 메타 누락률 |
| **Pitfalls** | 태그 과다/불일관, 권한 태깅 누락 |

### 7) SplitExtract (분해/추출하다, 1→N) ← 신설

| 항목 | 내용 |
|------|------|
| **정의** | 하나의 대상에서 독립 아티팩트들을 분리/추출. 디지털=M2(Structure), 물리=M1(Decompose) |
| **포함** | PDF 페이지 분할, 문서→표/이미지 추출, 로그에서 필드 추출, 어셈블리 분해, DB 정규화로 테이블 분리 |
| **제외** | 추출 후 결합은 Integrate, 라벨만 붙이면 Structure |
| **DoD** | 추출물 세트 + 매핑(원본↔추출) + 커버리지/정확도 지표 |
| **KPI** | 추출 정·재현율, 누락/중복률, 참조무결성 유지율 |
| **Pitfalls** | 참조 깨짐, 키 관리 실패, 컨텍스트 손실 |

## A.4 TRANSFER (전달·이동) — 5종

**목적:** 시·공간 좌표(접근성/적시성) 최적화

**Primary Mechanism:** M3 — Physical Coordinates (SpatialOptimize / TemporalOptimize)

### 1) Transport (운송하다)

| 항목 | 내용 |
|------|------|
| **정의** | 물리 자산을 장소 A→B로 실제 이동 (M3-SpatialOptimize) |
| **포함** | 교재/장비/키트 배송·회수, 반품 물류 |
| **제외** | 디지털 링크/파일 전달 → Distribute |
| **DoD** | 운송장/인수증, 도착·손상 여부 기록 |
| **KPI** | 리드타임, 손상·분실률, SLA 준수율 |
| **Pitfalls** | 포장 미흡, 반품 라우팅 혼선, 체인오브커스터디 누락 |

### 2) Distribute (배포하다)

| 항목 | 내용 |
|------|------|
| **정의** | 문서/콘텐츠/소프트웨어/이벤트를 대상에게 전달·릴리즈 (M3-TemporalOptimize) |
| **포함** | 제안서/설문 발송, 릴리즈 배포, 캘린더 초대 생성/리마인드, 시스템 간 동기화(API/Webhook, ETL Push) |
| **제외** | 서명·권리 발생 → COMMIT, 내용 편집 → TRANSFORM |
| **DoD** | 발송/릴리즈/초대 로그(도달·오픈·응답) |
| **KPI** | 도달률, 오픈/응답률, 타이밍 적합도, 큐 지연 |
| **Pitfalls** | 스팸 필터, 수신자/버전 오류, 아이덴티티 매핑 실패 |

### 3) HandOver (인계하다)

| 항목 | 내용 |
|------|------|
| **정의** | 책임/소유(Owner, Assignee, Custody)를 사람·팀·시스템 간 넘김 (M3-TemporalOptimize) |
| **포함** | 컨설턴트→운영팀 업무 인계, 티켓 Assignee 변경, 물리 자산 Custody 체인지 |
| **제외** | 단순 참조 공유(책임 이동 無), 권한 부여(IAM) → COMMIT.Authorization |
| **DoD** | 인계 요청/수락 로그, 소유자 필드 변경, 티켓/상태 전이 |
| **KPI** | 핑퐁 횟수, 대기시간, 재작업률, 첫-컨택 해결률 |
| **Pitfalls** | DoD/책임 범위 불명확, 수신확인 누락, 인계 기준·SLA 미정 |

### 4) Deliver (납품하다)

| 항목 | 내용 |
|------|------|
| **정의** | 합의된 산출물을 고객/내부 수신자에게 공식 납품 (M3-TemporalOptimize) |
| **포함** | 결과물 납품, 납품 패키지(산출물+메타/릴리즈노트) 전달 |
| **제외** | 검수/수락 결정 → ASSURE/COMMIT, 단순 배포 → Distribute |
| **DoD** | 납품 전송 증빙(전달 경로/시간/무결성 체크섬 등), 수령 확인(Received) |
| **KPI** | 첫 납품 적시율, 납품 후 반려(리턴) 비율 |
| **Pitfalls** | 수락 기준 미합의, 릴리즈 아티팩트 불일치, 증빙 미수집 |

### 5) Settle (정산하다)

| 항목 | 내용 |
|------|------|
| **정의** | 돈/금융자산의 실제 이동(수납/환불/청산) (M3-TemporalOptimize) |
| **포함** | 입금 확인, 환불 처리, 수수료/차지백 정리 |
| **제외** | 인보이스/세금계산서 생성 → TRANSFORM.Convert, 가격·대금 조건 합의 → COMMIT.Contract |
| **DoD** | 결제/수납 레코드(승인코드/금액/시각), 원장 반영 |
| **KPI** | DSO, 실패율, 수수료, chargeback 비율 |
| **Pitfalls** | 중복 청구, 환불 지연, 참조 무결성 깨짐, 조정분 미계상 |

## A.5 COMMIT (권리·의무 설정) — 6종

**Primary Mechanism:** M4 – Social (RightsAffirmation)

### 1) ExecuteContract (계약 체결하기)

| 항목 | 내용 |
|------|------|
| **정의** | MSA/SOW/계약서에 서명·조건 확정하여 법적 효력 발생 |
| **포함** | 전자서명 완료하기, 부속조건 확정하기 |
| **제외** | 제안서/계약 초안 작성 → TRANSFORM, 문서 발송 → TRANSFER |
| **DoD** | 서명본 PDF, 계약 레코드(상대방/발효일/버전) |
| **KPI** | 체결률, 사이클타임, 분쟁률 |
| **Pitfalls** | 상이본 서명, 부속조건 누락, 서명 권한 미확인 |

### 2) PromulgatePolicy (정책/표준 공표하기)

| 항목 | 내용 |
|------|------|
| **정의** | 할인·취소·품질·보안 등 규칙/표준을 공식화·공지 |
| **포함** | 환불 정책 공표하기, 품질 기준 고시하기 |
| **제외** | 룰 초안·기준표 설계 → DECIDE.SpecifyRules |
| **DoD** | 정책 문서(발효일/적용범위/버전) 및 공지 기록 |
| **KPI** | 준수율, 예외 승인율, 개정 주기 |
| **Pitfalls** | 공지 누락, 중복·충돌 정책, 법규 불일치 |

### 3) GrantAuthorization (권한/역할 부여·회수하기)

| 항목 | 내용 |
|------|------|
| **정의** | IAM에서 접근권/승인권/역할을 부여·변경·회수 |
| **포함** | 폴더 접근권 부여하기, 결재 권한 부여·회수하기 |
| **제외** | 열람만 허용(권리 변경 無) → 제외 |
| **DoD** | 권한 변경 로그(주체/범위/만료), 접근 검증 기록 |
| **KPI** | 승인 리드타임, 오남용률, 권한 크리프 |
| **Pitfalls** | 최소권한 원칙 위반, 퇴직자 권한 미회수 |

### 4) CommitResources (예산 확정·PO 발행·정원/용량 예약하기)

| 항목 | 내용 |
|------|------|
| **정의** | 예산·발주·좌석/용량을 시스템에 최종 확정·선점 |
| **포함** | 예산 확정하기, PO 발행하기, 좌석/용량 예약 확정하기 |
| **제외** | 배분안·스케줄안 작성 → DECIDE.Allocate/Schedule |
| **경계** | PO가 계약 성립 수단이면 ExecuteContract로 분류 |
| **DoD** | 예산/PO/예약 확정 레코드(금액/수량/기간) |
| **KPI** | 집행률, 변경 횟수, 잔액/노쇼율 |
| **Pitfalls** | 승인 체계 우회, 중복 예약, 이중 지출 |

### 5) Certify (사실/자격 인증하기)

| 항목 | 내용 |
|------|------|
| **정의** | 수료/성적/브랜드 보증 등 공식 인증을 발급·취소 |
| **포함** | 수료증 발급하기, 보증서 발급·무효화하기 |
| **제외** | 평가 채점·데이터 기록 → ASSURE/TRANSFORM |
| **DoD** | 인증서/자격 레코드(유효기간/취소 정책/검증수단) |
| **KPI** | 발급 정확도, 무효화율, 검증 응답률 |
| **Pitfalls** | 이름/ID 오기, 자격 요건 미검증 발급 |

### 6) RegisterFile (신고·허가·등록하기)

| 항목 | 내용 |
|------|------|
| **정의** | 관할 기관에 신고/허가/등록하여 법적 지위·권한 취득 |
| **포함** | 개인정보 처리 신고하기, 라이선스 갱신하기, 상표/특허 출원 등록하기 |
| **제외** | 신청서·부속 문서 작성 → TRANSFORM |
| **DoD** | 접수증/허가증, 요구조건 충족 증빙(체크리스트 포함) |
| **KPI** | 승인 리드타임, 보완 요청률, 갱신 누락률 |
| **Pitfalls** | 기한 초과, 요구 포맷 불일치, 증빙 누락 |

## A.6 ASSURE (보증·통제) — 6종

**목적:** 상태/품질/진위/준수의 확인·보증. 수정 행위는 분리.

**Mechanism Primary:** M4 – Social (StateAssurance)

### 1) Verify (규격/스펙 준수 여부를 검증하다)

| 항목 | 내용 |
|------|------|
| **정의** | 형식·무결성·규격 일치 여부 체크. M4(StateAssurance) |
| **포함** | 체크리스트 실행, 링크/가격/서명 필드 검증, QA 스모크 |
| **제외** | 발견된 오류 수정 → TRANSFORM–Repair |
| **DoD** | 검증 로그(항목별 합/불), 불합격 목록 |
| **KPI** | 결함 검출률, 허위 합격률(false pass) |
| **Pitfalls** | 표본 편향, 검증–수정 혼합 |

### 2) Validate (목적 적합성·효용을 검증하다)

| 항목 | 내용 |
|------|------|
| **정의** | 효용/전환/만족 등 목적 달성 여부 평가. M4(StateAssurance) |
| **포함** | 파일럿, A/B/N 테스트, 사용자 유효성 인터뷰(해석은 최소화) |
| **제외** | 상업 배포 → TRANSFER–Distribute |
| **DoD** | 결과 리포트(지표·통계·해석·한계) |
| **KPI** | 효과크기, 신뢰구간/유의성(p-value), 샘플 파워 |
| **Pitfalls** | p-hacking, 외삽(일반화) 오류 |

### 3) Monitor (지표를 상시 추적하고 알림을 발생시키다)

| 항목 | 내용 |
|------|------|
| **정의** | SLI/SLO·NPS·응답률 등 지속 추적/알람. M4(StateAssurance) |
| **포함** | 대시보드, 경보 규칙·임계치, 알림 라우팅 |
| **제외** | 알람 이후 의사결정/수정 → DECIDE / TRANSFORM |
| **DoD** | 알림/이벤트 로그, 임계치 변경 이력 |
| **KPI** | 탐지 지연(MTTD), 오탐/미탐, 가시성(커버리지) |
| **Pitfalls** | Alert fatigue, 임계치·소음 비율 튜닝 미흡 |

> **경계 규칙:** Monitor=상시·라인 주도, Audit=기간성·독립 주도.

### 4) Audit (절차·기록을 독립적으로 감사하다)

| 항목 | 내용 |
|------|------|
| **정의** | 재무/보안/프로세스를 독립 시각에서 사후 검증. M4(StateAssurance) |
| **포함** | 표본검사, 접근 로그 점검, 증빙 추적성 테스트 |
| **제외** | 시정조치 실행 → TRANSFORM / COMMIT |
| **DoD** | 감사보고서(범위·샘플링·증빙 링크·권고안), 경영진 브리핑 |
| **KPI** | 시정요청 수, 중대도 분포, 재발률 |
| **Pitfalls** | 독립성 훼손, 범위 누락/과도한 체크리스트 의존 |

### 5) AssessRisks (위험을 사전 식별·평가하다)

| 항목 | 내용 |
|------|------|
| **정의** | 시나리오/영향/완화책 사전 도출. M4(StateAssurance) |
| **포함** | FMEA, 위험 매트릭스(발생가능성×영향), 잔여 위험 평가 |
| **제외** | 통제의 정책화/공표 → COMMIT–Policy |
| **DoD** | 위험 목록(등급·소유자·기한)과 권고안 |
| **KPI** | 고위험 커버리지, 잔여 위험 추세 |
| **Pitfalls** | 가능성/영향 과소평가, 소유자·기한 미지정 |

### 6) IncidentResponse (사고에 대응·격리·소통하다)

| 항목 | 내용 |
